# Fix OOM Heap Crash — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Eliminare il crash OOM (JavaScript heap out of memory) causato da serializzazione dell'intero dataset + query N+1 + chiavi RSA in memoria

**Tech Stack:** Sails.js, sails-disk, Node.js streams (opzionale)

### Task 1: SyncCache — Export incrementale per modello

Il problema principale: `exportFromDB()` fa `find()` su 5 tabelle + `JSON.stringify()` dell'intero risultato ogni 5 secondi.

**Files:**
- Modify: `api/utility/SyncCache.js`

- [ ] **Step 1: Aggiungere dirty-tracking per modello**

In `SyncCache`, aggiungere un Set statico che traccia quali modelli sono stati modificati:

```javascript
// Dopo la riga const CACHE_PATH
static _dirtyModels = new Set();

static markDirty(modelName) {
  SyncCache._dirtyModels.add(modelName);
  SyncCache.scheduleSave();
}
```

- [ ] **Step 2: Riscrivere exportFromDB() per esportare solo i modelli dirty**

Invece di fare `find()` su tutte le tabelle, caricare il file cache esistente e sovrascrivere solo i modelli dirty:

```javascript
static async exportFromDB() {
  let existing = {};
  try {
    const raw = fs.readFileSync(CACHE_PATH, 'utf8');
    existing = JSON.parse(raw);
  } catch (e) { /* file non esiste, partire da zero */ }

  // IMPORTANTE: snapshot atomico + clear immediato per evitare race condition
  // Se nuovi markDirty() arrivano durante le find() async, finiscono nel prossimo ciclo
  const dirty = new Set(SyncCache._dirtyModels);
  SyncCache._dirtyModels.clear();
  if (dirty.size === 0) return existing;

  const modelMap = {
    organizzazioni: 'Organizzazione',
    strutture: 'Struttura',
    liste: 'Lista',
    assistiti: 'Assistito',
    assistitiListe: 'AssistitiListe',
  };

  for (const [key, model] of Object.entries(modelMap)) {
    if (dirty.has(model)) {
      const records = await sails.models[model.toLowerCase()].find();
      // Nota: sails-disk ignora .select()/.omit() — il delete post-query è l'unico modo
      existing[key] = records.map(r => {
        const o = { ...r };
        delete o.privateKey;
        return o;
      });
    }
  }

  return existing;
}
```

- [ ] **Step 3: Aggiornare `scheduleSave()` per usare il nuovo export**

Il metodo `scheduleSave()` (riga 135) già chiama `exportFromDB()` + `save()` — non serve modificarlo, solo verificare che funzioni col nuovo export incrementale.

### Task 3: api-strutture.js — Eliminare query N+1

Attualmente per ogni lista (5000!) fa 3 count + 1 find. Sostituire con una singola aggregazione.

**Files:**
- Modify: `api/controllers/api-strutture.js`

- [ ] **Step 1: Pre-aggregare contatori AssistitiListe**

Sostituire il loop N+1 (righe 50-74) con una pre-aggregazione:

```javascript
// PRIMA del loop sulle strutture, calcolare tutti i contatori in un colpo solo
const allAssistitiListe = await AssistitiListe.find()
  .select(['lista', 'stato', 'chiuso', 'dataOraIngresso', 'dataOraUscita']);

// Aggregare per lista
const statsByLista = {};
for (const al of allAssistitiListe) {
  const lid = al.lista;
  if (!statsByLista[lid]) {
    statsByLista[lid] = { inCoda: 0, usciti: 0, totale: 0, tempoTotaleMs: 0, tempoCount: 0 };
  }
  const s = statsByLista[lid];
  s.totale++;
  if (al.stato === 1 && !al.chiuso) {
    s.inCoda++;
  }
  if (al.chiuso) {
    s.usciti++;
    if (al.dataOraIngresso && al.dataOraUscita) {
      s.tempoTotaleMs += new Date(al.dataOraUscita) - new Date(al.dataOraIngresso);
      s.tempoCount++;
    }
  }
}

// Poi nel loop strutture, per ogni lista:
lista.stats = statsByLista[lista.id] || { inCoda: 0, usciti: 0, totale: 0 };
if (lista.stats.tempoCount > 0) {
  lista.stats.tempoMedioGiorni = Math.round((lista.stats.tempoTotaleMs / lista.stats.tempoCount) / 86400000 * 10) / 10;
}
```

Questo trasforma ~20000 query in **1 sola query** + un loop O(N) in memoria.

- [ ] **Step 2: Nota su memoria**

Nota: sails-disk ignora `.select()` — carica sempre l'intero record in memoria. Il vantaggio di questa riscrittura è puramente CPU/query: da ~20000 round-trip DB a 1 sola `find()` + loop O(N). Il peak memory resta invariato ma il tempo di risposta scende drasticamente.

// Helper per rimuovere chiavi RSA pesanti (~2.2KB per record)
function stripKeys(records) {
  return records.map(r => {
    const o = { ...r };
    delete o.privateKey;
    delete o.publicKey;
    return o;
  });
}

let assistiti = stripKeys(await Assistito.find());
